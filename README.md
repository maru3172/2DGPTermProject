게임의 간단한 소개

개발 내용

씬 종류: 타이틀 화면, 게임 화면, 게임 종료 화면

전환 방법

타이틀 → 게임: Enter 키를 눌러 전환.

게임 조작: 좌/우 키로 캐릭터 이동.

게임 → 게임 종료: 플레이어 사망 시 전환.

===================================================================

- 각 Scene 에 등장하는 GameObject 의 종류 및 구성, 상호작용

class Player: 이동 및 충돌 판정 제어.

class Enemy: 적의 이동 및 공격 제어.

class Bullet: 아군 및 적의 탄환 제어.

class Ground: 배경 그리기 및 제어.

- 함수 단위 설명

다음 발표에서 추가 예정.

===================================================================

- 사용한/사용할 개발 기법들에 대한 간단한 소개

게임 루프: 입력 처리, 업데이트, 렌더링을 반복하여 실시간 진행.

충돌 판정: AABB 기법으로 간단하고 효율적인 충돌 감지.

애니메이션 처리: 스프라이트 애니메이션으로 부드러운 움직임 표현.

입력 처리: 키보드와 마우스 입력으로 캐릭터 행동 제어.

자원 관리: 이미지를 효율적으로 로드하여 성능 최적화.

===================================================================

- 게임 프레임워크

게임 루프: 사용자 입력 처리, 상태 업데이트, 충돌 확인, 화면 렌더링을 통해 실시간으로 게임 진행.

씬 관리: 타이틀, 게임, 게임 종료 씬을 원활하게 전환하고 독립적으로 동작.

객체 지향 프로그래밍: 플레이어, 적, 총알을 클래스 형태로 정의하여 재사용성과 유지보수성 향상.

이벤트 시스템: 플레이어 사망, 적 출현 등 이벤트에 따라 게임 상태 변경.

자원 관리 모듈: 필요한 리소스를 효율적으로 로드하고 사용 후 메모리에서 해제.
